Extend

Extending JMVC could not be easier. Basically You can either use the JMVC.extend function which manages automatically the JMVC namespace , either write into the JMVC namespace (with some care) or create Your own ns.
Once You wrote your extension, save it within the app/extensions/ folder, require and use it in the Controller action.

If You save Your extension in /app/extensions/foo/mylib.js the right path to load it with the require function is 'foo/mylib'

There are some good reason to use the JMVC.extend function:

  • Take care of namespace collision, if not explicitly requested it will forbid You to override an existing element
  • Allows to do some check as far as the extension is required, nullifying the ns population if some conditions are not met
  • Allows to execute some arbitraty code as far as the extension is loaded, but before using it
