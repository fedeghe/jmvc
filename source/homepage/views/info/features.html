Features

  • $fr$

    One global variable.

  • $fr$

    Supports all alive browsers.

  • $fr$

    Includes many useful utilities: ajax, dom manipulation, event binding, etc...

  • $fr$

    Default `index` Controller and action (configurable).

  • $fr$

    Easily add aliases to actions.

  • $fr$

    Each View, Model, Controller or Interface inherits from built in constructors.

  • $fr$

    In views you can write special variable placeholders in a View to render another View on the fly or to assign a value to a View`s variable:

    • $myvar$ for substitute a variable that you set to the View instance through the set method
    • {{my_view}} for loading the content of my_view in place of that
    • {{my_view name=`Federico` surname=`Ghedina`}} as previous but here You can add on the fly variables to my_view, use $name$ and $surname$ in it, and find them replaced at render time.

  • $fr$

    Dinamically load other scripts and stylesheets, both parsable, so they can depend on Your variables.

  • $fr$

    The content produced by default affects the content of the body tag, but You can choose to affect the content of any node which can be found by the tiny minimal dummy selection engine.

  • $fr$

    Optimize Extensions dependencies using the JMVC.require function.

  • $fr$

    Automatic get extra parameters from url.

  • $fr$

    Add action routes in the Controller (see the demo controller).

  • $fr$

    Easily extend the JMVC object with your code (see any extension).

  • $fr$

    Allow any file extension You prefer in the url... I know You noticed the .jmvc extension in this page url.

  • $fr$

    Handle 404 Status for requests not matching a route rendering an error View and redirecting after three seconds to the default route.

  • $fr$

    Easily enque callbacks to some predefined hooks :onBeforeRender, onAfterRender, onBeforeParse, onAfterParse (i18n works thanks to a hook).

  • $fr$

    If a foo actions is dispatched JMVC will seek for a action_foo function in the controller, allowing private functions within the controller.

  • $fr$

    If a foo actions is dispatched, some hard named function if found will be executed:

    • before : called before any action
    • before_foo : called immediately before calling action_foo
    • now action_foo is called
    • after_foo : called immediately after calling action_foo
    • after : called after any action
    • action : if found in the controller, will be called if foo_action is not in the controller, as parameters will be passed the name of the view (foo here) and a literal containing all found parameters

  • $fr$

    Use placeholders even in js and css to substitute your own variables or the special preset variable $baseurl$ (this can be useful for absolute path to images, or for chunks of style rules shared among more selectors) or $rendertime$ (just to have an idea of the whole time spent from dispatching to page render)

  • $fr$

    many ready to use extensions, wrappers and widgets.

  • $fr$

    and MUCH MORE!
