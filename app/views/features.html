$fr$ One global variable.
$fr$ Supports all alive browsers
$fr$ Includes many useful utilities: ajax, dom manipulation, event binding, etc...
$fr$ Load views and models (synchronous ajax requests, otherwise model eval could be late) from a controller.
$fr$ Default `index` controller and action (configurable).
$fr$ Easily add aliases to actions
$fr$ Each view, model or controller has his own hash as private registry to contain variables.
$fr$ In views you can write special variable placeholders for other views or for view`s variable:
  • $myvar$ for substitute a variable that you set to the View instance through the set method
  • {{my_view}} for loading the content of my_view in place of that
  • {{my_view name=`Federico` surname=`Ghedina`}} as previous but here You can add on the fly variables to my_view, use $name$ and $surname$ in it, and find them replaced at render time.
$fr$ Dinamically load other scripts and stylesheets.
$fr$ The content produced by default affects the content of the body, but You can choose to affect the content of any node with an id attribute. You can edit the head with some limitations using JMVC.head methods (or your scripts).
$fr$ File sizes:
  • 13 KB for jmvc.pack.js (base62 encode, shrinked variables)
  • 37 KB for jmvc.js (development version, full commented)
$fr$ Total warranty for completely seo unfriendly pages :D
$fr$ Optimize modules dependencies with a require dummy function, .... no (AMD)ceremonies
$fr$ Automatic extra parameters extraction from url (a/d/y/u/5/...).
$fr$ Automatic extra parameters extraction from querystring (?id=3e3&t=rre&...).
$fr$ Add action routes in the controller (see test controller).
$fr$ Easily extend the JMVC object with your code (see any extension).
$fr$ Allow .html extension or any extension You prefer
$fr$ Handles 404 Status for requests not matching a route rendering an error view and redirecting after three seconds to the default route
$fr$ Use placeholders even in js and css to substitute your own variables or the special preset variable $baseurl$ (this can be useful for absolute path to images, or for chunks of style rules shared among more selectors) or $rendertime$ (just to have an idea of the whole time spent from dispatching to page render)
