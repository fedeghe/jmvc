• $fr$

    One global variable.

  • $fr$

    Supports all alive browsers.

  • $fr$

    Includes many useful utilities: ajax, dom manipulation, event binding, etc...

  • $fr$

    Load views and models (synchronous ajax requests, otherwise model eval could be late) from a controller.

  • $fr$

    Default `index` controller and action (configurable).

  • $fr$

    Easily add aliases to actions

  • $fr$

    Each view, model or controller has his own hash as private registry to contain variables.

  • $fr$

    In views you can write special variable placeholders for other views or for view`s variable:

    • $myvar$ for substitute a variable that you set to the View instance through the set method
    • {{my_view}} for loading the content of my_view in place of that
    • {{my_view name=`Federico` surname=`Ghedina`}} as previous but here You can add on the fly variables to my_view, use $name$ and $surname$ in it, and find them replaced at render time.

  • $fr$

    Dinamically load other scripts and stylesheets.

  • $fr$

    The content produced by default affects the content of the body, but You can choose to affect the content of any node which can be found calling the tiny minimal selection engine.

  • $fr$

    Edit the head using JMVC.head methods (or your scripts).

  • $fr$

    Files sizes:

    • 11.5 KB jmvc.min.js (gzipped and packed version, base62 encode, shrinked variables)
    • 24.6 KB jmvc.js (gzipped, development version, fully commented)

  • $fr$

    Optimize modules dependencies using the JMVC.require function.

  • $fr$

    Automatic extra parameters extraction from url.

  • $fr$

    Add action routes in the controller (see test controller).

  • $fr$

    Easily extend the JMVC object with your code (see any extension).

  • $fr$

    Allow any extension You prefer

  • $fr$

    Handles 404 Status for requests not matching a route rendering an error view and redirecting after three seconds to the default route

  • $fr$

    Easily add callbacks to some predefined hooks :onBeforeRender, onAfterRender, onBeforeParse, onAfterParse (i18n works thanks to a hook)

  • $fr$

    Use placeholders even in js and css to substitute your own variables or the special preset variable $baseurl$ (this can be useful for absolute path to images, or for chunks of style rules shared among more selectors) or $rendertime$ (just to have an idea of the whole time spent from dispatching to page render)
